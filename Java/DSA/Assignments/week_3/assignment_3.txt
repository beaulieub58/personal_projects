(5.10) What are the advantages and disadvantages of the collision resolution strategies you used in 5.1

1. Separate Chaining:

    Advantages of separate chaining resolution are most relevant in a table with a load factor ~= 1, or the number 
    of elements is roughly equal to the number of indices. Still, seprate chainining isn't as vunerable to increased load factors as open addressing is.
    
    Collisions are resolved by inserting new values at the beginning of the hashed index's bucket, which is done 
    in constant time, when each bucket index contains a linked list or any other ADT that supports preppending. 

    Lastly, since insertions at each bucket index are occuring within a separate ADT, resizing the original table size
    is not neccessary as each element can be inserted in the ADT per bucket.

    Disadvantages of separate chaining include extra complexity in setup and memory usage due to allocating new ADT objects
    for each index in the table. 

    Also, if hashed values do cluster in to a concentrated number of buckets, searching for those values could
    be O(N) due to the need to traverse longer list ADT's in each bucket.

2. Linear Probing

    The advantages of linear probing are that it is relatively easy to implement, as finding an available index for insertion
    after a collision is identified consists of incrementing the hashed valued by 1 and modulating by the list size.

    Linear probing also keeps all hashed values in a simple array which results in less memory usage.

    Disadvantages include primary clustering, which occures when blocks of occuoped cells start forming. This can lead
    to longer insertion times due to more attempts needed to resolve the initial and subsequent collisions. 

    To add to that thought, we see performance degrade nearly exponentially as the load factor of the table increases.
    As the load factor grows, with linear probing, the number of probes necessary to resolve a collision increases signicantly. 
    If a table is expected to be more than half full, linear probing can be problematic or larger amount of memory are needed to
    build arrays large enough to accommodate the expected number of insertions, which negates one of the strategy's primary advantages.

3. Quadratic Probing

    Quadratic probing has the advantage of eliminating the primary clustering, and is relatively easy to implement compared to seaprate chaining

    A considerable disadvantage to quadratic probing is the lack in gaurantee that all elements to be inserted will find
    a cell to be inserted at, which is exacerbated if the table's length is not a prime number. This is due to fact that
    at most, half of the table can be used as alternative locations to resolve collisions. This is known as secondary clustering.

4. Double Hashing

    Advantages of double hashing include the elimination of primary and secondary clustering, which theoretically should
    reduce the number of probes necessary to resolve collisions. 

    If a secondary hash function is strong and compliant with the table's prime size, double hashing is more performant.

    However, the caveat is double hashing requires the double hash function allow all table cells to be probed. If the table size is not prime,
    we can run in to the same problems secondary clustering presents. Alternative locations could prematurely run out.

    An additiona disadvantage is the fact that we have to implement a hash function twice in this use case, which can be expensive
    when dealing keys like strings.
