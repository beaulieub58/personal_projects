Recursion:
a recursive function is one that is defined in terms of itself. The function call recalls the function at some point in its definition
must include a base case where the function will terminate without having to induce the recursion
each recursive function call is bookkept by the computer.
some values in recursion will cause a base case to never be reached.
In recursive programming, there's not such thing as a special case because the recruvsive call will work up or back to the base case an potentially make a call to invalid parameters along the way
Design Rule: Recursive calls work correctly.
Four rules:

1. Base Case: you must have a base case that can be solved without recursion
2. Making progress: the recursive calls you make in the function must work towards the base case
3. All function calls work correctly, assumption
4. Never duplicate work by solving the same instance of a problem in additional recursive calls
